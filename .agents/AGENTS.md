# Agent Rules

The following rules apply to all tasks and interactions in this project:

1. **Do not push without permission:** Never run `git push` or execute push commands without explicit, written permission from the user.
2. **Do not implement without asking:** Do not begin code modifications, file creations, database refreshes, or any actual implementation steps without first asking the user and obtaining their consent.
3. **Clarify instead of assuming:** When a request is given, if there is any ambiguity, confusion, or lack of specification, ask clarifying questions instead of implementing or pushing code. Always explain your plans and wait for user confirmation.
4. **Do not modify or reset the backend database without consent:** Never run database refreshes, migrations, seeds, or commands that modify backend state without explicit warning and consent from the user.
